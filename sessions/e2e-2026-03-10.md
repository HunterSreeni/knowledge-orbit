## Bugs Found

1. **[Critical] `useSupabaseUser()` returns null in write page `save()` despite valid auth cookie**
   - **Route:** /write
   - **Step:** Save Draft click
   - **Error shown:** "Not signed in. Please refresh and try again."
   - **Observed:** Auth cookie `sb-iqbsxrxlexplcaqixggy-auth-token` present in browser. Reload does not fix it.
   - **Root cause:** `useSupabaseUser()` reactive ref is not populated when the save function runs. This is a Nuxt Supabase SSR/hydration timing issue — the user ref is null at setup time when the page is server-side-rendered or navigated to directly after login.
   - **Impact:** All draft/publish saves fail. The entire write flow is broken for authenticated users.
   - **File:** `app/pages/write.vue:52` — `if (!user.value?.id)`

2. **[Critical] Admin middleware redirects admin user to `/` instead of allowing access to /admin/***
   - **Routes affected:** /admin, /admin/ads, /admin/analytics (all admin subroutes)
   - **Expected:** Admin panel loads
   - **Actual:** Redirect to / (homepage)
   - **Root cause:** Same `useSupabaseUser()` hydration issue. Admin middleware at `app/middleware/admin.ts:3` checks `if (!user.value) return navigateTo('/login')`. Since user.value is null, it hits the next check: queries `profiles` table for role. Because `user.value` is null on line 9, the `.eq('id', user.value.id)` would throw or return no rows, and `data?.role !== 'admin'` → redirects to `/`.
   - **DB confirmed:** `profiles` record for admin user has `role = 'admin'` — so this is purely a middleware timing/hydration issue, not a data issue.
   - **File:** `app/middleware/admin.ts`

1. **Fix Bug #1 (Critical — blocks write flow):** In `app/pages/write.vue`, `useSupabaseUser()` is called at setup root but the value is null when `save()` executes. Options:
   - Use `await useSupabaseClient().auth.getUser()` directly inside `save()` for a fresh user fetch rather than relying on the reactive ref.
   - Or ensure the Nuxt Supabase plugin resolves user before the page renders (check `@nuxtjs/supabase` config for `redirect` and `cookieName` options).

Consider fetching user via `useSupabaseClient().auth.getUser()` rather than `useSupabaseUser()` inside middleware, or adding a `watchEffect`/`await` to ensure the user is resolved before the middleware check.
