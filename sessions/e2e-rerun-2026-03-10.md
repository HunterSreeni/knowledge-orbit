## Remaining Issues

### Issue 1 — Post-save navigation routes to public post view (NEW / pre-existing)
- **Severity**: Medium
- **Steps**: Login → /write → fill title + content → Save Draft
- **Observed**: URL changes to `/posts/[slug]`, H3Error "Post not found" thrown (console), 406 from Supabase (`status=eq.published` query finds nothing for a draft). The write editor layout remains visible (Nuxt appears to handle the error softly), but there is no success toast or redirect to dashboard.
- **Expected**: Save Draft should show a success toast and stay on /write (or redirect to /dashboard), not navigate to the public post URL.
- **Note**: The actual save IS working — the insert succeeds. This is a post-save routing bug in the write page component.

### Issue 2 — Dashboard shows 0 posts despite posts existing in Supabase (NEW / possibly pre-existing)
- **Severity**: Medium
- **Steps**: Login as admin → save a draft → navigate to /dashboard
- **Observed**: Dashboard shows "All (0)", "Drafts (0)", "Published (0)" and "No posts yet" even though the draft was confirmed saved in Supabase.
- **Expected**: Dashboard should show the saved draft under "Drafts (1)".
- **Likely cause**: RLS policy on `posts` table may require `author_id = auth.uid()` but the insert may not be setting `author_id` correctly, or the dashboard query filters by a profile `id` rather than Supabase `auth.uid()`.

### Issue 3 — Admin overview shows 0 for all stats (informational / expected for empty DB)
- **Severity**: Low (informational — no posts, users, or likes have been created via the public UI yet)
- **Observed**: 0 Total Posts, 0 Published, 0 Users, 0 Total Likes
- **Expected**: Would show 1 draft if Issue 2 is resolved.
